mac命令行

  • 用vscode写github blog：vscode中下载Markdown Preview Enhanced插件编辑md文件，cmd+k v快捷键可以打开，还可以自己配置背景。

  • vscode + hexo写github blog中的坑（转自 简书作者溜逗）
    博客内无法显示图片（转自 简书作者溜逗）

    1.cd到博客根目录下 查看_config.yml文件 查找 post_asset_folder 字段确定post_asset_folder设置为true -> post_asset_folder:true

    2.设置 post_asset_folder 参数后，在建立文件时，Hexo 会自动建立一个与文章同名的文件夹，您可以把与该文章相关的所有资源都放到此文件夹内，这样就可以更方便的使用资源。

    3.到博客的根目录下执行 npm install https://github.com/CodeFalling/hexo-asset-image --save命令来进行插件的安装。

    4.然后创建一文章 hexo new "test" 然后查看博客的 ../source/_posts目录下的文件，会看到存在一个test 文件夹 和 test.md 文件，将图片放在test 文件夹下

git入门

Posted on 2018-04-19

git init

  • 在我们新建一个空仓库后需要初始化这个仓库
    1.首先新建一个项目目录demo
    2.命令行进入demo下
    3.git init在demo中创建一个.git目录，她就是一个仓库

git add

  • 此时在demo下创建一个文件后，只是创建成功，并没有保存
    1.touch 1.txt创建一个txt文件
    2.git status -sb查看状态，发现新建文件前为？
    3.此时用git add 1.txt添加进暂缓区
    4.再次git status -sb发现已存入

git commit -v

  • 使用git commit -m“信息” 将你 add 过的内容「正式提交」到本地仓库，并添加一些注释信息，方便日后查阅
    1.git commit -v 提交时显示所有diff信息
    2.完成后才是正式提交上

命令入门

Posted on 2018-04-17

下面介绍几种常用的命令

mv

mv [源路径] [目的路径]：移动节点
将1.txt改名为2.txt,后缀可改
mv 1.txt 2.txt 

touch

touch ：创建文件
创建一个1.txt的文件
touch 1.txt

ls

ls[选项] [目录或文件名] ：查看路径
ls -a :显示不隐藏的文件与文件夹
ls -l :显示不隐藏的文件与文件夹的详细信息
ls -al: 显示所有文件与文件夹的详细信息

cat

cat [选项] [文件]：连接文件或标准输入并打印
cat 文件名：查看文件内容
cat xx.txt | grep yy：查找文件内容
cat >b.txt << abc ：编辑b.txt并以abc结束

如何使用 explainshell 这个网站

首先google翻墙
直接打开 explainshell.com
搜索框中输入你想要查询的命令
网页中会显示命令的解释和用法
