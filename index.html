Posted on 2018-04-24

http介绍

HTTP（HyperText Transfer Protocol超文本传输协议）的作用就是指导浏览器和服务器如何进行沟通。

常用端口：80
浏览器发出请求
服务器响应
浏览器下载响应内容

http请求

创造请求：

请求的格式

1.请求最多包含四部分，最少包含三部分。（也就是说第四部分可以为空）
2.第三部分永远都是一个回车（\n）
3.动词有 GET POST PUT PATCH DELETE HEAD OPTIONS 等
4.这里的路径包括「查询参数」，但不包括「锚点」
5.如果你没有写路径，那么路径默认为 /
6.第 2 部分中的 Content-Type 标注了第 4 部分的格式

chrome查看请求内容

1.右键检查->network->Hide data URLs
2.进入一个网址
3.点击最上方的一条记录->Request Hearders->view source

4.即可看到命令行中显示的请求内容

http响应

响应实例

响应格式

1 协议/版本号 状态码 状态解释
2 Key1: value1
2 Key2: value2
2 Content-Length: 17931
2 Content-Type: text/html
3
4 要下载的内容
  • 状态码(服务器对浏览器)
    • 1xx 不常用
    • 2xx 成功
      1. 200 OK
        请求已成功，请求所希望的响应头或数据体将随此响应返回。在GET请求中，响应将包含与请求的资源相对应的实体。在POST请求中，响应将包含描述或操作结果的实体。
      2. 204 No Content
        服务器成功处理了请求，没有返回任何内容
    • 3xx 不存在
      1. 301 Moved Permanently
        被请求的资源已永久移动到新位置
      2. 302 Found
        临时不存在，可能还会回来
      3. 304 Not Modified
        和上次的回应相同，不回应
    • 4xx 客户端错误
      1. 404 Not Found
    • 5xx 服务端错误
      1. 500 Internal Server Error
        通用错误消息，服务器遇到了一个未曾预料的状况，导致了它无法完成对请求的处理。没有给出具体错误信息。
      2. 502 Bad Gateway

chrome查看响应

1.打开 Network
2.输入网址
3.选中第一个响应
4.查看 Response Headers，点击「view source」，点击「view source」，点击「view source」你会看到响应的前两部分
5.查看 Response 或者 Preview，你会看到响应的第 4 部分

命令入门

Posted on 2018-04-17

下面介绍几种常用的命令

mv

mv [源路径] [目的路径]：移动节点
将1.txt改名为2.txt,后缀可改
mv 1.txt 2.txt 

touch

touch ：创建文件
创建一个1.txt的文件
touch 1.txt

ls

ls[选项] [目录或文件名] ：查看路径
ls -a :显示不隐藏的文件与文件夹
ls -l :显示不隐藏的文件与文件夹的详细信息
ls -al: 显示所有文件与文件夹的详细信息

cat

cat [选项] [文件]：连接文件或标准输入并打印
cat 文件名：查看文件内容
cat xx.txt | grep yy：查找文件内容
cat >b.txt << abc ：编辑b.txt并以abc结束
